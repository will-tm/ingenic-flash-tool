# ingenic_flash (compact C flasher)

A minimal C reimplementation of `ingenic-flash-tool`'s `detect` and `flash`
(SPI NOR) paths, sized for an embedded Linux host built with Buildroot. Only
runtime dependency is **libusb-1.0**. Code is ~350 lines; the binary is ~460 KB,
almost all of which is the embedded SPL/U-Boot blob.

Scope (by request): **T33ZN only, SPI NOR only.** The board firmware
(`ginfo`/`spl`/`uboot`), the cfg1/cfg2 payloads, the T-series memory map, and
the SPI-NOR chip parameters are all baked in at build time from the Python
tree via `gen_blob.py` into `firmware_blob.h`. Several NOR chips can be embedded
at once; the tool reads the live JEDEC ID and selects the matching cfg2.

## Build (host)

```sh
make            # regenerates firmware_blob.h if missing, then compiles
./ingenic_flash detect
```

`firmware_blob.h` is generated by `gen_blob.py`, which imports the in-tree
`ingenic_flash` Python package and runs its real `patch_cfg2` — so the bytes
sent to the burner are identical to the Python tool's default NOR path.

## Usage

```sh
ingenic_flash detect
ingenic_flash flash <image> [--offset N] [--erase-all] [--no-reboot] [-v]
```

`flash` boots the device (ROM → SPL → stage2 burner), checks the live JEDEC ID
against the chip the blob was built for (`0xEF7018`, W25Q128), and streams the
image in 64 KB CRC'd chunks. Default is per-sector erase; `--erase-all` does a
full-chip erase first.

## Targeting NOR chips

By default `gen_blob.py` embeds the **entire** `spiflashinfo.cfg` database
(every supported vendor/part). The tool reads the live JEDEC at flash time and
picks the matching embedded cfg2; an unrecognized chip is refused with the list
of supported IDs. Chips >16 MB automatically get 4-byte-address opcodes.

```sh
python gen_blob.py            # embed the whole DB (default), then: make
python gen_blob.py 0xef7018 0xef4020   # or restrict to a subset
```

Each chip adds ~2 KB. The full DB (~54 chips) costs ~110 KB of binary, on top
of the ~440 KB shared SPL/U-Boot blob.

## Buildroot integration

Copy `package/ingenic-flash/` into your Buildroot tree (or a `br2-external`),
add `source "package/ingenic-flash/Config.in"` to your package menu, and enable
`BR2_PACKAGE_INGENIC_FLASH`. It selects `libusb` automatically.

For a **local checkout** (no remote), override the source in your
`local.mk` / br2-external:

```make
INGENIC_FLASH_OVERRIDE_SRCDIR = /path/to/ingenic-flash-tool
```

The package builds `embedded/ingenic_flash` and installs it to
`/usr/bin/ingenic_flash`. `firmware_blob.h` is committed, so the target build
needs no Python — just the cross toolchain, libusb, and pkg-config.

To run without root on the target, add a udev rule (or run as root):

```
SUBSYSTEM=="usb", ATTR{idVendor}=="a108", MODE="0666"
```

## Not ported (vs. the Python tool)

- NAND / eMMC flashing (NOR only)
- Runtime SPI-NOR auto-detection from `spiflashinfo.cfg` (baked in instead)
- `--gpio` boot-ROM GPIO poking (e.g. PMIC power-hold) — add if your board
  needs a line asserted to stay powered through the flash
- Multi-chip / multi-board selection
